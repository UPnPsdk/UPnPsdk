15// Socket module
16// =============
17// Responsibility: Manage ONE network IPv4 and IPv6 stream or datagram socket.
18//
19// This module mainly consists of the CSocket class but also provides free
20// functions to manage a socket. The problem is that socket handling isn't very
21// good portable. There is different behavior on the supported platforms Unix,
22// MacOS and Microsoft Windows. The CSocket class atempts to be consistent
23// portable on all three platforms by using common behavior or by emulating
24// missing functions on a platform.
25//
26// Specification for CSocket
27// -------------------------
28// The class encapsulates and manages one raw socket file descriptor. The file
29// descriptor of a valid socket object cannot be changed but the object with
30// its immutable file descriptor can be moved and assigned to another socket
31// object. Copying a socket object isn't supported because having two objects
32// with the same file descriptor may be very error prone in particular with
33// multithreading. Effort has been taken to do not cache any socket information
34// outside the socket file descriptor. All socket informations are direct set
35// and get to/from the operating system with the file descriptor. The socket
36// file descriptor is always valid except on an empty socket object.
37//
38// --- empty socket object ---
39// An empty socket object can be instantiated with the default constructor,
40// e.g. 'CSocket sockObj;'. It is a valid object and should be destructed. When
41// moving a socket object, the left over source object is also empty. An empty
42// socket object has an INVALID_SOCKET defined and no valid content. It throws
43// an exception if using any of its Setter and Getter. Moving and assigning it
44// is possible.
45//
46// --- address family ---
47// Only address family AF_INET6 and AF_INET is supported. Any other address
48// family throws an exception.
49//
50// --- socket type ---
51// Only SOCK_STREAM and SOCK_DGRAM is supported. Any other type throws an
52// exception.
53//
54// --- valid socket file descriptor ---
55// I get this from the C standard library function
56// int ::socket(address_family, socket_type, protocol).
57// Other arguments than address family and socket type do not instantiate a
58// valid socket object and throw an exception. For the protocol argument is
59// always the default one used that is internal hard coded with argument 0.
60//
61// --- option SO_REUSEADDR ---
62// I don't set the option to immediately reuse an address of a local listening
63// socket after it was closed. Instead I respect the 'TIME_WAIT'. This is a
64// security issue as described at
65// REF:_[Bind:_Address_Already_in_Use](https://hea-www.harvard.edu/~fine/Tech/addrinuse.html).
66// I reset SO_REUSEADDR with constructing a socket object on all platforms if
67// it should be set by default. This is unclear on WIN32. See next note.
68//
69// --- option SO_EXCLUSIVEADDRUSE on Microsoft Windows ---
70// THIS IS AN IMPORTANT SECURITY ISSUE! Lock at
71// REF:_[Using_SO_REUSEADDR_and_SO_EXCLUSIVEADDRUSE](https://learn.microsoft.com/en-us/windows/win32/winsock/using-so-reuseaddr-and-so-exclusiveaddruse#application-strategies)
72// I always set this option with constructing a socket object on a WIN32
73// platform.
74// --Ingo

215 // * This flag can only be set on sockets of address family AF_INET6.
216 // * It is always false on a socket with address family AF_INET.
217 // * It is always true on Unix platforms after binding a socket to an
218 // address of family AF_INET6 if passive mode isn't set on the address
219 // info (flag AI_PASSIVE).
220 // * With an address info set to passive listen on local addresses (flag
221 // AI_PASSIVE) IPV6_V6ONLY can be modified before binding it to an

223 // listen to IPv6 and IPv4 connections if IPV6_V6ONLY is set to false.
